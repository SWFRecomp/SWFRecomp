Implementation follows Copy-on-Store pattern:
1. StringAdd concatenates on stack (preserves optimization)
2. SetVariable copies stack string to heap (new allocation)
3. GetVariable copies heap string to stack (for operations)
4. Variable owns heap memory (freed on reassignment/exit)
